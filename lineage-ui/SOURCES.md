## 誠實記錄的取捨

1. **卷軸的白/祝福/詛咒無法100%確認對應官方物品**:研究 6sword.com 的 RSC 資料庫(`rawEtcitems`
   272筆)與 NC 官方 CDN(`assets.playnccdn.com/.../ItemIcon/{編號}.png`),
   確認 itemId 25(對武器施法的卷軸,CDN圖468)/26(對盔甲施法的卷軸,CDN圖467)是官方「一般強化卷軸」
   本尊;但天堂系列的「祝福/詛咒」實為道具的**狀態屬性**、不是獨立圖示(inven.co.kr 社群文亦證實
   「축복/저주 받은 아이템」是屬性而非另一張圖)。因此祝福卷/詛咒卷沒有另外的官方CDN圖示可查,
   改用董事長截圖裡三個花紋明顯不同的羊皮卷(白/花草紋/血紅裂紋)當視覺代表——是「真實遊戲截圖」,
   但不保證跟遊戲內部真正的祝福/詛咒卷軸圖示逐一對應。

2. **格線沒有用真分隔紋理逐格貼**:`grid-divider.png` 裁下來確認可用(乾淨的十字交叉),
   但它是單一角度/單一位置的樣本,若要在 `.invgrid` 上重複貼齊,tile 的像素間距是固定值,
   跟我們響應式(92vw、依螢幕寬度變動)的格子邊界無法保證每個尺寸都對齊,
   在390寬手機以外的寬度上很可能跑版、格線對不上格子角。改成保留原本已驗證能在390寬正常運作的
   CSS bevel(亮/暗雙色border)畫格線,只把石紋背景換成真材質——「不跑版」優先於「每根線都是照片」。

3. **上/下薄邊蓋沒有裁真圖**:來源截圖的頂/底邊很薄、且被黃字彈幕/工具列圖示切碎,
   裁不出一段乾淨、可重複的水平石紋,改用CSS金繩收尾(顏色比色自 `frame-rail.png` 的繩紋色),
   跟左右真柱銜接處肉眼看基本無縫。

4. **CDN官方圖(468/467)最終沒用在卷軸格上**:雖然驗證了官方「一般強化卷軸」CDN連結存在,
   但那是單色系「羊皮卷+武器/盔甲剪影圖騰」的通用款,跟截圖裡三種花紋不同的卷軸相比,
   截圖裁的版本跟遊戲實際畫面(董事長要的「1:1」基準)更貼近,故優先採用截圖版本。
   CDN連結留在此檔備查:`https://assets.playnccdn.com/static-lineageclassic-gamedata/resources/ItemIcon/468.png`(武器)、
   `.../467.png`(盔甲)。
